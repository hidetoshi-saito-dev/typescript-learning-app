# 定着支援機能の設計（達成バッジ・連続学習・進捗グラフ・復習導線）

作成: 2026-06-12 ／ 正本。実装 PR とセット。

## 概要

「解いて終わり」から「続けたくなる・戻ってきたくなる」への一歩。ホームの進捗カード直下に
**学習のきろく**パネルを追加し、4要素を表示する:

1. **達成バッジ**: はじめの一歩 / 初級・中級・上級マスター / 全レッスン制覇（5種）
2. **連続学習（ストリーク）**: 🔥 N日連続
3. **進捗グラフ**: 直近8週の週別レッスン完了数（手書き SVG ではなく CSS バー）
4. **復習導線**: 完了が最も古いレッスン最大3本への「もう一度解く」リンク

## なぜこの設計か

### DB スキーマ変更ゼロ（最重要の判断）
- ログインユーザー: `lesson_progress.completed_at`（既存カラム）から**すべて導出できる**
  - バッジ＝完了集合の集計 / ストリーク＝完了日の連続性 / グラフ＝週別バケット / 復習＝最古完了順
- ゲスト: 既存の localStorage（完了 ID 配列）に加えて、**活動ログ用キーを新設**
  （`ts-learning-guest-activity-v1`・`{ id, d: "YYYY-MM-DD" }` の配列・初完了時のみ追記）
  - 過去の完了分には日付がないため、**ゲストのストリーク/グラフは導入日以降の記録から育つ**
    （バッジ・復習は日付不要なので過去分も即動く）。この割り切りを UI 文言でも示す
- 本番 Supabase への migration・RLS 変更・新規テーブルが不要＝**リリースリスクが最小**

### 「完了イベント」への正規化
ログイン（DB 行）とゲスト（活動ログ）を `{ lessonId, date }` の **DatedCompletion** に正規化し、
ストリーク・グラフ・復習を**純関数**（`src/lib/progress/insights.ts`）で計算する。
- 純関数に切り出す理由: vitest で境界（日付の連続・週バケット・タイムゾーン）を固定するため

### 日付はクライアントのローカルタイムゾーンで確定する
`completed_at` は UTC。サーバー（Vercel = UTC）で日付文字列化すると日本のユーザーの「今日」とずれて
ストリークが破綻する。**日付化はクライアント側でのみ行い、ハイドレーション前はプレースホルダを表示**
（既存のゲスト進捗と同じ「hasHydrated まで実値を出さない」パターン）。これは SSR/CSR の
ハイドレーション不一致対策でもある。

### ストリークの定義（仕様の明文化）
- 「**初完了**が1件以上ある日」を学習日とする（復習のみの日は対象外＝完了データから導出できる範囲）
- 連続は**今日または昨日**で終わる並びを数える（今日まだ解いていなくても昨日までの連続は生きている）
- 上記が将来「復習も数えたい」となったら活動ログの記録契機を広げる（その時もスキーマ変更不要）

### 表示しない判断
- ランキング・他者比較はしない（個人学習アプリ・比較は離脱要因になり得る）
- グラフはゲストでデータ0件のあいだは出さず、1件たまったら出す（空グラフの寂しさ回避）

## 詳細

- `src/lib/progress/insights.ts`: `toLocalDateString` / `computeStreak(dates, today)` /
  `weeklyCounts(dates, today, weeks=8)`（月曜起点・古→新） / `computeBadges(completed, items)`
- `src/lib/progress/guest.ts`: `recordGuestActivity`（markComplete 内で初完了時のみ追記）・
  `getGuestActivitySnapshot`（useSyncExternalStore 用・参照安定化）
- `src/lib/progress/actions.ts`: `getServerProgressDetail(): { lessonId, completedAt }[]`
  （`getServerCompletedLessons` はこれに置き換えて削除）
- `src/components/home/RetentionPanel.tsx`: 表示本体（client）。LessonList の進捗カード直下に配置
- バッジ5種: 🌱 はじめの一歩(1+) / 🎓 初級マスター(15) / 🏅 中級マスター(16) / 🏆 上級マスター(8) /
  👑 全レッスン制覇(39)。未獲得はグレースケール表示（次の目標が見える）

## 学びポイント（オーナー向け）

- **「機能追加＝テーブル追加」ではない**。既存データから導出できるものを先に出し切ると、
  リリースリスクとメンテ面積を増やさずに体験を足せる
- タイムゾーンは「保存は UTC・表示と日付判定はローカル」。日付の確定をどちら側でやるかを
  決めずに作ると、ストリークのような「日付の同一性」に依存する機能は必ず壊れる
- useSyncExternalStore＋hasHydrated パターン（既存のゲスト進捗）は「クライアントでしか
  決まらない値」全般に再利用できる
