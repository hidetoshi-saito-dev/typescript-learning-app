# デザインブリーフ: レッスン（エディタ）画面

## 目的

ユーザーがTypeScriptコードを書き、即座にフィードバックを受け取る。
MVPのコア体験。この画面の使いやすさがサービス価値の大半を決める。

---

## ユーザーストーリー

- 初学者として、問題文を読んでコードを書き、型エラーを日本語で教えてもらいたい
- 初学者として、正解したら次のレッスンに自然に進みたい
- 初学者として、詰まったときにヒントを見て自力で解決したい

---

## 受け入れ条件

- [ ] 問題文（Markdown）が左ペインに表示される
- [ ] TypeScriptエディタが右ペインに表示され、入力できる
- [ ] 型エラーがある場合、エディタ内に赤波線が表示される
- [ ] 型エラーがある場合、エラーパネルに日本語の説明が表示される
- [ ] 型エラーがない場合、エラーパネルは表示されない
- [ ] 「答え合わせ」ボタンを押すとハイブリッド判定が実行される
- [ ] 正解時: 正解アニメーション + 「次のレッスンへ」ボタン表示
- [ ] 不正解時: 失敗したテストケースの日本語メッセージ表示
- [ ] 「ヒント」ボタンで段階的ヒントを1つずつ表示できる
- [ ] モバイルでは縦積みレイアウトになる

---

## レイアウト仕様

### デスクトップ（lg以上）

```
[ヘッダー: ← コースへ戻る | レッスン N/10 | ヒント | 解答を見る]
+---------------------------+------------------------------------+
| 問題ペイン（40%）         | エディタペイン（60%）              |
|                           |                                    |
| ## レッスンタイトル       | 1  // 型をつけてください           |
|                           | 2  const fruits =                  |
| 解説テキスト（Markdown）  | 3    ['apple', 'banana']           |
|                           |                                    |
| 【問題】                  | [エラーパネル: 型エラーがある時]   |
| ... 課題の説明 ...        |                                    |
+---------------------------+------------------------------------+
[アクションバー: [答え合わせ]                    次のレッスン →]
```

### モバイル（lg未満）

問題ペイン（折りたたみ可）→ エディタ → エラーパネル → アクションバー の縦積み

## コンポーネントリスト

> Props型の正本は Notion「🧩 コンポーネントリスト」。下表は要点の抜粋であり、差異がある場合は正本を優先する。（2026-05-26 M2対応）

| コンポーネント | Props型（要点） | 備考 |
|--------------|----------------|------|
| `LessonLayout` | `lesson: Lesson` | 2カラムグリッド全体 |
| `ProblemPane` | `title, description: string` | Markdownレンダリング |
| `CodeEditor` | `initialCode, onChange, errors` | エディタ本体 |
| `ErrorPanel` | `errors: TypeScriptDiagnostic[]` | エラー0件なら非表示 |
| `JudgeButton` | `onJudge, disabled` | 答え合わせボタン |
| `HintDrawer` | `hints, isOpen, onClose` | 段階的ヒント |
| `JudgeResult` | `result: JudgeResultType` | 正解/不正解パネル |

---

## 設計メモ

- **「答え合わせ」は明示的ボタン**: タイピング中の自動判定はUXが悪い（エラーだらけの画面になる）
- **エラーパネルは条件表示**: 型エラーがない間は下ペインを広く使えるようにする
- **ヒントは段階表示**: 一気に答えを見せない。TypeHeroと同じ設計
- **エディタ背景はダーク**: コード記述に集中しやすく、目の疲れを軽減
