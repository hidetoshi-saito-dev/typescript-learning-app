# 学習UXパターンの将来候補（業界比較メモ）

記録: 2026-06-12（実践クラス設計レビュー R-1/R-2/R-4 より）／ステータス: アイデア保管

## 本設計（実践クラス）で業界水準を満たした点

- 受け入れ条件の事前表示 = Codecademy のチェックポイント方式と同型（条件と判定が 1:1）
- 「回帰テストに守られて既存コードを変更する」体験 = 商用でも稀有な差別化要素
- 連作の worked example → completion problem（faded guidance）は学習科学的に正当
  （初学者には worked example が有効・習熟者は問題解決優位＝expertise reversal。
  卒業制作クラスは③主体・②最小アンカーで自由度を確保しており整合）
- 既存の復習導線（古い順3本）× 連作 = 間隔反復と整合

## 将来候補（今回スコープ外・フィールド/UI追加を伴う）

1. **段階的ヒント**: hint の `\n\n` ブロックを「もう少しヒント」ボタンで段階開示する UI。
   Exercism / Execute Program 等の定番。現 UI は全開示のみ。
2. **模範解答の事後公開**: 正解後に模範解答＋解説を表示（Exercism のコミュニティ解答文化の縮小版）。
   連作では「次レッスンの initialCode」が実質それを担っているが、最終レッスン（042/045/048）には次がない。
3. **所要時間フィールド**: 今回は scenario 文中の「目安: 15分」で代替（文言規約）。
   構造化するならフィールド追加＋一覧表示が候補。
